## 简介

OpenClaw Manager 是 OpenClaw AI 网关的图形化管理工具。本指南将帮助您了解如何使用 OpenClaw Manager 完成从安装到日常管理的所有操作。

### 适用读者

本指南适用于：
- 初次使用 OpenClaw 的新用户
- 需要图形化管理界面的系统管理员
- 希望简化配置流程的开发者

### 前置知识

使用 OpenClaw Manager 不需要深入了解命令行操作，但建议具备以下基础知识：
- 了解 AI 模型的基本概念
- 熟悉 API 密钥的使用
- 了解基本的网络配置（如代理设置）

## 安装

### 下载与安装

请参考 [README.md](../README.md#安装) 中的安装说明，根据您的操作系统选择适合的安装方式。

### 首次启动

安装完成后，启动 OpenClaw Manager：

1. **macOS**: 从 Applications 文件夹启动，或在 Spotlight 中搜索 "OpenClaw Manager"
2. **Windows**: 从开始菜单或桌面快捷方式启动
3. **Linux**: 运行 `openclaw-manager` 命令或点击应用图标

### 初始化向导

首次启动时，应用会显示欢迎界面和初始化向导：

1. **语言选择** - 选择您偏好的界面语言
2. **安装 OpenClaw** - 选择是否立即安装 OpenClaw
3. **配置检查** - 系统会自动检查环境依赖

## 功能详解

### 仪表盘

仪表盘是 OpenClaw Manager 的主界面，提供服务状态概览和快捷控制。

#### 服务状态概览

仪表盘显示以下关键信息：

**安装状态**
- OpenClaw 是否已安装
- 当前安装的版本号
- 安装路径

**服务控制**
- **启动服务**: 点击启动 OpenClaw 服务
- **停止服务**: 点击停止运行中的服务
- **重启服务**: 快速重启服务以应用配置更改

**状态指示器**
- 🟢 绿色: 服务运行中
- 🔴 红色: 服务已停止
- 🟡 黄色: 服务启动中或异常

#### 系统资源监控

仪表盘显示实时系统资源使用情况：
- **CPU 使用率**: 显示当前 CPU 使用百分比和核心数
- **内存使用**: 显示已用内存、总内存和可用内存
- **磁盘空间**: 显示数据目录的磁盘使用情况

#### 最近活动

显示最近的操作记录：
- 安装/升级记录
- 配置更改
- 服务启停
- 错误告警

#### 诊断告警

如果系统检测到问题，会在仪表盘显示告警卡片：
- 环境配置问题
- 依赖缺失
- 网络连接问题
- 可一键修复的问题

### 模型配置

模型配置模块用于配置和管理 OpenClaw 使用的 AI 模型。

#### 添加模型

**步骤 1: 选择提供商**

支持的模型提供商：
- OpenAI (GPT-4, GPT-3.5)
- Anthropic (Claude)
- Azure OpenAI
- Google (Gemini)
- 本地模型 (Ollama, LM Studio)
- 自定义 API

**步骤 2: 配置连接**

- **API 端点**: 模型的 API URL
- **API 密钥**: 访问模型的认证密钥（将安全存储）
- **模型名称**: 具体的模型标识符

**步骤 3: 设置参数**

- **温度 (Temperature)**: 控制输出的随机性 (0-2)
- **最大令牌数**: 单次请求的最大输出长度
- **Top P**: 核采样参数
- **频率惩罚**: 降低重复词的概率
- **存在惩罚**: 降低重复主题的概率

**步骤 4: 高级选项**

- **重试次数**: 请求失败时的重试次数
- **超时时间**: 请求超时时间（秒）
- **并发限制**: 同时进行的请求数限制

#### 模型列表

模型列表显示所有已配置的模型：

| 列 | 说明 |
|----|------|
| 名称 | 模型的显示名称 |
| 提供商 | 模型提供商 |
| 状态 | 连接状态（在线/离线/错误） |
| 操作 | 编辑/删除/测试 |

#### 测试连接

添加或编辑模型后，建议测试连接：
1. 点击模型行中的 **"测试"** 按钮
2. 系统会发送一个简单的请求验证连接
3. 显示测试结果和响应时间

#### 模型优先级

可以设置模型的优先级顺序：
1. 在模型列表中拖动调整顺序
2. 优先级高的模型会被优先使用
3. 当高优先级模型不可用时，自动切换到下一个

### Agent 管理

Agent 管理模块用于管理 OpenClaw 的智能代理，控制它们的启用状态。

#### Agent 列表

显示所有已配置的 Agent：

| 列 | 说明 |
|----|------|
| 名称 | Agent 的显示名称 |
| ID | Agent 的唯一标识符 |
| 描述 | Agent 的功能描述 |
| 状态 | 启用/禁用状态 |
| 操作 | 开关控制 |

#### 启用/禁用 Agent

每个 Agent 都有独立的开关控制：

**启用 Agent**
1. 在 Agent 列表中找到目标 Agent
2. 点击开关切换为 **"开启"** 状态
3. Agent 将立即生效，可接收和处理请求

**禁用 Agent**
1. 在 Agent 列表中找到目标 Agent
2. 点击开关切换为 **"关闭"** 状态
3. Agent 将停止接收新请求

#### 查看 Agent 详情

点击 Agent 卡片可查看详细信息：
- Agent ID 和名称
- 功能描述
- 当前状态
- 配置参数（只读）

#### 设置当前 Agent

可以指定一个 Agent 作为当前默认 Agent：
1. 在 Agent 列表中找到目标 Agent
2. 点击 **"设为当前"** 按钮
3. 该 Agent 将成为默认使用的 Agent

**注意**: Agent 的详细配置（创建、编辑提示词、配置能力等）需要通过 OpenClaw 的配置文件或 API 进行。Agent 管理界面主要用于快速启用/禁用和切换当前 Agent。

### 技能商店

技能商店（Skill Store）允许您浏览、安装和管理 OpenClaw 的 AI 技能。

#### 技能市场

**浏览技能**
1. 进入 **"技能商店"** 页面
2. 浏览分类或搜索特定技能
3. 查看技能的详细信息、评分和下载量

**技能分类**
- 代码辅助
- 数据分析
- 文本处理
- 图像处理
- 自动化工具
- 其他

#### 安装技能

**从市场安装**
1. 在技能市场中找到需要的技能
2. 点击 **"安装"** 按钮
3. 等待下载和安装完成
4. 安装成功后技能自动启用

**更新技能**
- 系统会自动检查已安装技能的更新
- 有可用更新时显示 **"更新"** 按钮
- 点击即可升级到最新版本

#### 管理已安装技能

**查看已安装技能**
- 切换到 **"已安装"** 标签页
- 查看所有已安装的技能列表
- 显示版本号、状态和最后更新时间

**启用/禁用技能**
- 在技能卡片上切换开关
- 禁用后技能不可用但保留配置

**配置技能**
- 点击技能的 **"配置"** 按钮
- 修改技能特定的设置参数
- 保存配置后即时生效

**卸载技能**
- 点击 **"卸载"** 按钮
- 确认卸载操作
- 可选择是否保留配置数据

#### 技能详情

点击技能卡片查看详细信息：
- 技能名称和描述
- 版本信息和更新日志
- 作者信息
- 使用说明和文档链接
- 用户评价和评分

### 诊断修复

诊断修复模块帮助排查和解决 OpenClaw 运行中的问题，提供一键修复功能。

#### 系统检查

**环境检查**
- 操作系统版本兼容性
- 必要的系统组件
- 磁盘空间充足性
- 内存可用性

**依赖检查**
- OpenClaw 二进制文件
- 配置文件完整性
- 数据目录权限
- 网络连接状态

**安装检查**
- OpenClaw 是否正确安装
- 版本兼容性
- 环境变量配置

#### 网络诊断

**连接测试**
- 测试与模型提供商的连接
- 检查代理设置是否正确
- 测试 DNS 解析
- 检查防火墙设置

**性能测试**
- 测量到各模型的延迟
- 测试带宽和吞吐量
- 检查连接稳定性

#### 一键修复

对于检测到的常见问题，提供一键修复功能：

**可修复的问题**
- 配置文件损坏
- 权限不足
- 缺失的依赖项
- 错误的网络配置
- 服务启动失败

**修复步骤**
1. 点击 **"运行诊断"** 进行全面检查
2. 查看检测到的问题列表
3. 勾选要修复的问题
4. 点击 **"一键修复"**
5. 查看修复结果和日志

#### 诊断结果

诊断完成后显示：
- **通过项**: 显示绿色勾选标记
- **警告项**: 显示黄色感叹号，可能需要关注
- **错误项**: 显示红色叉号，需要修复

每个问题都包含：
- 问题描述
- 严重程度（信息/警告/错误）
- 是否可自动修复
- 修复建议

#### 导出诊断报告

可以将诊断结果导出为报告：
- 点击 **"导出报告"** 按钮
- 选择保存位置
- 报告包含系统信息、配置、诊断结果和建议

### 日志查看

日志查看模块提供实时日志流式查看功能，支持多源日志聚合。

#### 日志源管理

**查看日志源**
- 显示所有可用的日志源
- 包括 OpenClaw 服务日志、系统日志等
- 显示每个日志源的大小和路径

**添加日志源**
1. 点击 **"添加日志源"**
2. 输入日志文件路径
3. 指定日志源名称
4. 确认添加

**移除日志源**
- 在日志源列表中点击移除按钮
- 确认移除操作

#### 实时日志流

**WebSocket 流式传输**
- 日志实时更新，无需手动刷新
- 支持高并发日志写入
- 自动重连机制

**日志级别筛选**
- DEBUG: 详细的调试信息
- INFO: 一般信息
- WARN: 警告信息
- ERROR: 错误信息
- 可多选级别进行组合筛选

**搜索功能**
- 实时搜索日志内容
- 支持关键词高亮
- 搜索结果导航

#### 日志操作

**自动滚动**
- 默认自动跟随最新日志
- 向上滚动时暂停自动滚动
- 滚动到底部恢复自动滚动

**清空显示**
- 点击 **"清空"** 按钮清空当前显示
- 不影响实际日志文件

**导出日志**
- 选择导出格式（文本/JSON/CSV）
- 设置时间范围和筛选条件
- 选择保存位置

#### 日志统计

显示日志统计信息：
- 总日志大小
- 日志源数量
- 各级别日志数量分布

#### 检查更新

**自动检查**

## 高级功能

### 自动更新

OpenClaw Manager 支持自动检查更新：

**配置自动更新**
1. 打开 **设置 > 更新**
2. 启用 **"自动检查更新"**
3. 选择更新通道（稳定版/测试版）
4. 配置自动安装或手动确认

### 备份与恢复

**创建备份**
1. 打开 **设置 > 备份**
2. 点击 **"创建备份"**
3. 选择备份内容（配置/数据/全部）
4. 选择备份位置

**恢复备份**
1. 点击 **"恢复备份"**
2. 选择备份文件
3. 选择恢复内容
4. 确认恢复操作

### 多实例管理

对于高级用户，可以管理多个 OpenClaw 实例：

**添加实例**
1. 点击 **"添加实例"**
2. 输入实例名称和路径
3. 配置实例特定的设置

**切换实例**
- 在实例列表中选择要管理的实例
- 界面会切换到该实例的配置

### 命令行集成

OpenClaw Manager 提供命令行接口：

```bash
# 启动应用
openclaw-manager

# 启动特定实例
openclaw-manager --instance "production"

# 以服务模式启动（无界面）
openclaw-manager --daemon

# 执行命令
openclaw-manager --exec "start-service"
```

### 常见问题

- 验证配置文件语法

- 检查网络连接
- 确认模型提供商服务状态
